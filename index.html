__proto__: что это?

  • Служебное свойство объекта
  • В спецификации называется [[Prototype]]
  • (!) не путать с prototype
  • Содержит либо указатель на объект, либо null
  • Может отсутствовать в объекте
  • Невозможно установить как обычное свойство

Object.create(prototypeOrNull)

var a = {}; это var a = Object.create(Object.prototype)

var a = []; это var a = Object.create(Array.prototype)

var a = new Test(); это var a = Object.create(Test.prototype)

// a === {__proto__: prototypeOrNull}

__proto__: как работает?

			val testObj = {
			 exampleField: 10
			};
			console.log(testObj.exampleField)

__proto__: как работает?

			console.log(getField(testObj, "exampleField")
			function getField(object, fieldName) {
			    do {
			        if (fieldName in object) {
			            return object[fieldName];
			        } else {
			            object = object.__proto__;
			        }
			    } while (object && typeof object === 'object');
			    return undefined;
			}

__proto__: зачем?

Позволяет задать одно "хранилище" методов и свойств для схожих объектов.

prototype: что это?

  • Свойство объекта-функции
  • Содержит что угодно

prototype: откуда берётся?

Создаётся автоматически при создании функции (не стрелочной).

По умолчанию указывает на объект, содержащий свойство constructor, равное самой функции.

			function test() {}
			// test.prototype = {
			//     constructor: test
			// }

prototype: как работает?

При использовании оператора new.

			var a = new F();

			function F() {
